Prompting と RAG

September 12, 2024

学んだ知識を自分なりに調べてまとめてみます。今日のテーマはプロンプティングとRAGについてです。

Promptとは

prompt is a natural language request submitted to a language model to receive a response back. Prompts can contain questions, instructions, contextual information, few-shot examples, and partial input for the model to complete or continue.
（DeepL翻訳）プロンプトとは、言語モデルに自然言語のリクエストを送信し、応答を返すことである。これには質問、指示、文脈情報、例など、モデルが完了または続行するための部分的な入力を含めることができます。 (Google Cloud)

つまり、AIにタスクを指示する命令文のことをプロンプトと言います。
この命令文により、AIの出力文のクオリティが変わります。その質を向上させる方法が大きく分けて2種類あります。

1. プロンプトで工夫する方法
2. 外部ツールを利用（RAG）する方法

ちなみに、プロンプトを研究する分野として Prompt Engineering と Prompt Design があります。どちらも近い存在ですが、違いは以下の通りです。

“prompt design focuses on formulating effective instructions, prompt engineering fine-tunes and enhances AI models to better respond to those prompts.”
（DeepL翻訳）プロンプト・デザインは効果的な指示の策定に焦点を当て、プロンプト・エンジニアリングはそれらのプロンプトによりよく反応するようにAIモデルを微調整し、強化する。
(Medium)

1. プロンプトで工夫する方法

Few-shot Prompting

(arXiv)

プロンプトに複数の例を与える方法です。スクリーンショットにあるように、仮にある文章を英語からフランス語に翻訳したいとします。翻訳することを指示したうえで、具体的な例をプロンプトに明記してあげます。そして最後に翻訳してほしい単語を伝えることで、事前に渡した事例の法則にしたがい、AIが精度の高い回答を出すというものです。

zero-shot - 例を入れない
one-shot - 例を1つだけ提示
few-shot - 例を複数提示

グラフによると、複数の例を入力することで回答の精度が上がると報告されています。ちなみに、このようにAIが文脈から学習することを In-Context Learning と呼びます。

Fine Tuning との使い分けとしては、Fine Tune することで、プロンプトに毎回事例を入れる必要がなくなり、モデルも調整されているので性能が出やすいです。一方で、Fine Tune されていることでタスクによっては不向きなものがあり、汎用性がないのがデメリットです。

命令文の書き方に注目する

2022年12月の論文では、命令文の投げかけ方によって出力性能が変わるという検証結果が示されています。これは分類タスクにおいて、プロンプトの書き方で精度がどう変化したかを表したものです。(arXiv)

Chain-of-Thought (CoT) Prompting

Few-shot Prompting と少し似ていますが、CoT は事例とともに思考過程も入れるのが特徴です。CoT を使うことで、AIの出力にも思考過程を書いてくれるようになり、結果として、特にモデルサイズが大きい場合に推論タスクで性能が向上します。(NeurIPS)
追記：ChatGPT o1モデルも、まさにこの CoT を追求しています。

Zero-shot CoT（“Let's think step by step” の追加）

命令文の最後に “Let’s think step by step” と入れるだけで、思考過程を明示し、推論能力が上がるというものです。(NeurIPS)

2. 外部ツールを利用（RAG）する方法

RAG - Retrieval Augmented Generation

RAGとは、大規模言語モデル（LLM）によるテキスト生成に、外部情報の検索を組み合わせることで、回答精度を向上させる技術のことを指します。
→ AI特有のハルシネーションを防ぐのに便利です。

RAGが簡単に体験できるプラットフォーム： NotebookLM

このように外部ツールや知識を利用して精度を上げるモデルを、Augmented Language Models（ALM）と呼びます。

Retrieval の仕組み

Retrieval の役割は、外部テキスト p から、こちらが投げた質問（クエリ）q に類似した文章を見つけることです。(link)

この類似度を測るにあたって、Sparse Retriever と Dense Retriever があります。

Sparse Retriever は特定の重要なキーワード（TF-IDF）が含まれているかどうかの判定を得意とし、Dense Retriever は文章の意味的（Embedding）な類似度を計算することを得意とします。(arXiv)
Dense Retriever（Neural Retriever）には、Cross-encoder・DPR・ColBERT などさまざまな種類が存在します。
