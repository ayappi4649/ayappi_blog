問題文A - Jiro (link)

A, B, C の三兄弟がいます。この 3 人の年齢関係は、3 つの文字 SAB, SAC, SBC によって与えられ、それぞれ以下を意味します。

- SAB が < の場合、A は B より年下であり、> の場合、A は B より年上である。
- SAC が < の場合、A は C より年下であり、> の場合、A は C より年上である。
- SBC が < の場合、B は C より年下であり、> の場合、B は C より年上である。

三兄弟の次男、つまり二番目に年上の人は誰ですか？

入力例

< < <

出力例

B

解説

あまりいい方法が思いつかず、全パターンを if 文で書きました。もっといい方法があるのでは……と思いつつ、まずは解けた形をメモしておきます。

N, M, C = map(str, input().split())

if N == "<" and M == ">" or N == ">" and M == "<":
    print("A")

if N == "<" and C == "<" or N == ">" and C == ">":
    print("B")

if C == "<" and M == ">" or C == ">" and M == "<":
    print("C")

例えば A が出力されるパターンを考えてみます。ABC という 3 人兄弟がいるので、次男が A となるパターンは、B < A < C または C < A < B の2通りです。

今回与えられる入力値は SAB, SAC, SBC で、それぞれ A・B・C の関係性が不等号で表されています。その関係性を先ほどの2通りと照らし合わせると、次のようになります。

① C < A < B のとき → SAB: "<"、SAC: ">"
② B < A < C のとき → SAB: ">"、SAC: "<"

ここでは SAB を N、SAC を M と定義しているので、コードでは以下の条件になります。

if N == "<" and M == ">" or N == ">" and M == "<":
    print("A")

問題文B - Taro (link)

AtCoder 王国では、長男に必ず「太郎」という名前を付けます。長男以外には「太郎」という名前は付けません。長男とは、各家で生まれた男の子のうち最も早く生まれた者を指します。

AtCoder 王国には N 戸の家があり、M 人の赤子が生まれました。また、M 人の赤子が生まれる前には、N 戸のどの家も赤子が生まれたことはありませんでした。

赤子の情報が生まれた時系列順に与えられます。

i 番目に生まれた赤子は、Ai 番目の家で生まれ、Bi が M のとき男の子、F のとき女の子です。

M 人の赤子それぞれについて、付けられた名前が「太郎」か判定してください。

入力例

2 4
1 M
1 M
2 F
2 M

出力例

Yes
No
No
Yes

解説

# Nが家の数、Mが赤子の人数
N, M = map(int, input().split())

first_son = [False] * (N + 1)

for _ in range(M):
    # Aが何番目の家で生まれたかを表し、Bは性別で、
    # MはMale、FはFemaleを表す。
    A, B = input().split()
    A = int(A)

    if B == "M" and not first_son[A]:
        print("Yes")
        first_son[A] = True
    else:
        print("No")

first_son = [False] * (N + 1) で、各家に「最初の男の子がまだ生まれていないか」を記録するフラグを用意しておきます。

(N + 1) となっているのは、Python のリストが 0 番目から始まるためです。

初期値では、家の数だけ False をリストに入れて、まだどの家にも男の子が1人もいない状態を作ります。

例えば N = 4 で 4 戸の家があるなら、初期値の first_son は次のようになります。

[False, False, False, False, False]

このうち実際に使うのは 1 番目から 4 番目までで、どの家にもまだ男の子が誕生していないことを表しています。

次に if 文で、上から順番に A 番目の家にすでに男の子がいるかを確認します。

A 番目の家の子が M（男の子）で、かつ first_son[A] が False の場合、それはその家で初めて生まれた男の子ということになるため、Yes を出力します。

例えば入力例では、最初に 1 番目の家で男の子が誕生しています。

1 M

このとき first_son[1] はまだ False なので、not first_son[A] が成り立ちます。よって、男の子かつ初めての男の子という条件を満たし、Yes が出力されます。

そして、その参照した False を True に変えることで、「A 番目の家にはすでに男の子が生まれた」と記録できます。

すると、また同じ家で男の子が誕生しても、if B == "M" and not first_son[A]: が成り立たなくなるため、No が出力されます。

例えば入力例では、そのあと 1 番目の家でさらに男の子が誕生しています。

1 M

この時点では first_son[1] がすでに True になっているため、not first_son[A] は成り立ちません。したがって、No が出力されます。
